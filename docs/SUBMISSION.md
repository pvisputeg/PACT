# PACT Build Week submission draft

## Proposed track

**Work & Productivity**

## One-line description

PACT is an Enterprise Outcome Operating System that turns consequential business signals into verified, independently challenged, human-authorized action and measurable outcomes.

## The flagship story

Operation Northstar begins when Copper Alloy C-17 for Aurelis Industrial Systems is diverted by 12 days. Northstar Plant 7 has 5.4 usable production days, 318 orders and 42 strategic customers at risk, and $8.7M of committed revenue exposed.

PACT verifies the signal before acting, traces the impact through a live synthetic plant twin, compares three bounded strategies, asks a separate Auditor to challenge Balanced Recovery, requires the Plant COO to approve a condition-bound Action Contract, demonstrates a deterministic unsafe-action rejection, executes ten commitments across eight functions, observes the plant state, and closes at 96.1% protected revenue for $389,000.

## Why it matters

Enterprise AI often ends at an answer, recommendation, or task list. PACT closes the gap between intelligence and accountability: target, evidence, dissent, authority, action, observation, closeout, and learning remain connected in one governed record.

## How Codex was used

Codex was the engineering partner for scenario design, contracts, domain logic, React experience, visual system, plugin packaging, MCP runtime, safety controls, tests, artifact generator, judge scripts, and documentation. The repository preserves these artifacts as inspectable evidence rather than treating Codex as a hidden code generator.

## How GPT-5.6 was used

The optional artifact generator uses the OpenAI Agents SDK with high reasoning and strict Zod structured outputs. A PACT Outcome Lead proposes an evidence-cited plan. A separate independent PACT Auditor receives an immutable packet and challenges assumptions. Both have zero business tools. Human authorization remains external, and deterministic code alone enforces safe execution.

The Strategy and Audit screens validate and display the accepted artifact’s recommendation, assumptions, counterfactual, response IDs, trace IDs, provider, and estimated cost. The free path uses a visibly labeled `LOCAL SCHEMA FIXTURE · NO API CALL`. Explicit genuine mode fails closed on malformed or stale output; automatic mode discloses fallback. The earlier genuine artifact belongs to a historical prototype and is never represented as Northstar evidence. A fresh genuine Northstar artifact requires intentional generation and retains its own response IDs, trace IDs, usage, and cost ledger.

## Accurate implementation statement

Implemented locally: Enterprise Mission Control, Northstar Plant 7 Digital Twin, investigation, Outcome Contract, ten-control Proofline, causal cascade, three strategies, strict visible model-artifact boundary, separate audit gate, human Action Contract, ten-node Action Graph, fifteen safe local MCP tools, changing twin state, five observations, 96.1% closeout, eight learning records, executive and technical ledger, twelve-stage replay, proof report, Codex plugin, GPT-5.6 generator, tests, and full judge verification.

Not implemented or claimed: production connectors, live enterprise data, external customer sending, autonomous approval, production deployment, causal proof, or financial ROI.

All organizations, people, evidence, transactions, results, and tool calls are synthetic.
